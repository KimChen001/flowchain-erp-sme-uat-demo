# Xingyun Reference Alignment v1

FlowChain 默认产品结构以 xingyun 的基础资料、采购、销售、库存、结算和打印对象为长期基准。一级导航固定为：首页、基础资料、采购管理、销售管理、库存管理、结算管理、报表中心、系统管理。AI、证据链、复核与数据治理只作为业务页辅助能力或“高级与内部”入口。

本轮实际检查了 xingyun `README.md`，`xingyun-basedata` 的供应商、客户、商品、分类、品牌、仓库、库位、物流公司、支付方式和打印模板控制器，`xingyun-sc` 的采购订单/收货/退货、销售订单/出库/退货、库存查询/流水/调整/预警/盘点/调拨控制器，`xingyun-settle` 的费用/预付款/对账/结算/应收应付项目控制器，以及打印模板服务、设置 BO/VO、`V1.14__print_template.sql`、采购订单/收货单/销售出库单的 `/print` 接口和 Print BO。

## 防跑偏规则

- 页面改名不等于对象已实现；只有存在真实字段、列表/详情和可操作入口才标记实现状态。
- 供应商资料留在基础资料；发票、费用、预付款、对账、结算和三单匹配留在结算管理。
- 交付风险与发货单分开，库存异常与调整单分开，批次/序列号与库存预警分开。
- 普通列表优先标题、简短副标题、操作、筛选、表格和详情；AI、证据链与复核放到详情次级区域。
