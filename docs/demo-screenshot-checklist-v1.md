# Demo Screenshot Checklist v1

Use this checklist for manual demo/UAT screenshot capture. No screenshot automation is required in this round.

## Core Screens

- Today Cockpit first viewport with summary cards and recommended actions.
- Today Cockpit inventory risk section.
- Today Cockpit procurement follow-up section.
- Recent procurement document table with clean typography.

## Inventory Evidence

- Inventory risk detail or focused SKU view.
- SKU recovery controls.
- Inventory movement entry point.
- Inventory exception entry point.
- PR draft preview entry from low-stock action.

## Procurement Evidence

- Procurement follow-up or PO detail.
- PO evidence with related PR/RFQ/GRN/invoice where available.
- RFQ detail or RFQ response status.
- GRN/receiving exception detail.
- Invoice or three-way-match exception visibility.

## Global Search

- Search result for a PO such as `PO-2026-1301`.
- Search result for a SKU such as `SKU-00287`.
- Evidence rows in the search result.
- Navigation recovery after opening a result.

## AI Assistant

- AI Assistant open on Today Cockpit.
- Answer to `今天最需要处理什么？`.
- AI answer with evidence cards.
- AI evidence click path where supported.
- Timeout fallback copy if manually simulated.

## Action Draft Review

- PR draft preview from inventory risk.
- RFQ draft preview from sourcing/quotation context.
- Supplier follow-up draft preview from procurement/supplier follow-up.
- Action Draft Review shell with payload, validation, evidence, and audit trail.
- Disabled/future confirmation state.

## Safety and Boundary Shots

- AI provider disabled/default safety explanation if surfaced in demo.
- Draft-first confirmation boundary.
- Evidence/navigation recovery controls.
- Typography-clean procurement or cockpit table.

## Optional Supporting Shots

- Master Data item/supplier context.
- SRM supplier operational context.
- Finance collaboration/invoice match visibility.
- Reports/imports/data management entry point if relevant to the audience.
